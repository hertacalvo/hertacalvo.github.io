2.1 String Concatenation and Creation of Temporary String Object

  • The following statement creates a new string by "adding" 
                  std::string my_line = "*" + std::string(first.size()+2,' ') + "*";
    • The expression std::string(first.size()+2, ' ') within this statement creates a temporary STL string but does not associate it with a variable.

2.2 Character Arrays and String Literals

  • In the line below "Hello!" is a string literal and it is also an array of characters (with no associated variable name). 
                  cout << "Hello!" << endl;
    • As char array can be initialized as: 
                  char h[] = {'H', 'e', 'l', 'l', 'o', '!', '\0'};
    or as 
                  char h[] = "Hello!";
    In either case, arrah h has 7 characters, the last one being the null character
  • The C language provides many functions for manipulating these "C-style strings". We don't study them much anymore because the "C++" STL string library is much more logical and easier to use.
  • One place we do use them is in file names and command-line arguments, which you will use in Homework 1.

Converesion Between Standard Strings and C-Style String Literals

  • We regularly conver/cast between C-style & C++-style (STL) strings. For example: 
                  std::string s1("Hello!");
              std::string s2(h);
    where h is defined above
  • You can obtain the C-style string from a standard string using the member funcion c_str , as in s1.c_str()
Sample Proof$$ \begin{align} T(0) &= \frac{0\cdot (0+1)}{2} = 0 && \text{Basis Step}\\ T(k) &= \frac{k\cdot (k+1)}{2} && \text{Inductive Hypothesis} \\ T(k+1) &= \frac{(k+1)(k+2)}{2} && \text{Inductive Step} \\ &= T(k) + k + 1 && \text{From Lemma 1}\\ &= \frac{k \cdot (k+1)}{2} + k+1 && \text{From I.H}\\ &= \frac{k \cdot (k+1)}{2} + \frac{2(k+1)}{2} \\ &= \frac{k^2 + k + 2k + 2}{2} \\ &= \frac{k^2 + 3k + 2}{2} \\ &= \frac{(k+1)(k+2)}{2} \end{align} $$

L-Values and R-Values

  • Consider the simple code below. String a becomes "Tom" . No big deal, right? 
                  std::string a = "Kim";
              std::string b = "Tom";
              a[0] = b[0];
    • Let's look more closely at the line: a[0] = b[0]; and think about what happens. In particular, what is the difference between the use of a[0] on the left hand side and b[0] on the right hand side?
  • Syntactically, they look the same. But,
    • The expression b[0] gets the cahr value, 'T' from the string location 0 in b. This is an r-value .
    • The expression a[0] gets a reference to the memory location associated with string location as an l-value .
    • The assignent operator stores the value in the referenced memory location.
  • The difference between an r-value and an l-value will be especially signficant to get operators later in the semester
  • What's wrong with the code? 
                  std::string foo = 'hello';
              foo[2] = 'X';
              cout << foo;
              'X' = foo[3];
              cout << foo;
    Your C++ compiler will complain with something like: " non-lvalue in assignment "
